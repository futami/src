  46 /**

  47  * BOARD_NETLIST_UPDATER

  48  * updates the #BOARD with a new netlist.

  49  *

  50  * The changes are made to the board are as follows they are not disabled in the status

  51  * settings in the #NETLIST:

  52  * - If a new component is found in the #NETLIST and not in the #BOARD, it is added

  53  *   to the #BOARD.

  54  * - If a the component in the #NETLIST is already on the #BOARD, then one or more of the

  55  *   following actions can occur:

  56  *   + If the footprint name in the #NETLIST does not match the footprint name on the

  57  *     #BOARD, the footprint on the #BOARD is replaced with the footprint specified in

  58  *     the #NETLIST and the proper parameters are copied from the existing footprint.

  59  *   + If the reference designator in the #NETLIST does not match the reference designator

  60  *     on the #BOARD, the reference designator is updated from the #NETLIST.

  61  *   + If the value field in the #NETLIST does not match the value field on the #BOARD,

  62  *     the value field is updated from the #NETLIST.

  63  *   + If the time stamp in the #NETLIST does not match the time stamp  on the #BOARD,

  64  *     the time stamp is updated from the #NETLIST.

  65  * - After each footprint is added or update as described above, each footprint pad net

  66  *   name is compared and updated to the value defined in the #NETLIST.

  67  * - After all of the footprints have been added, updated, and net names and

  68  *    pin function properly set, any extra unlock footprints are removed

  69  *    from the #BOARD.

  70  *

  71  */

  84      * @return true if process was completed successfully

  85      */
