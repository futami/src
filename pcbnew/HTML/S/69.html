 360     /**

 361      * function TransformPadsShapesWithClearanceToPolygon

 362      * generate pads shapes on layer aLayer as polygons and adds these polygons to aCornerBuffer

 363      * Useful to generate a polygonal representation of a footprint in 3D view and plot functions,

 364      * when a full polygonal approach is needed

 365      * @param aLayer = the layer to consider, or UNDEFINED_LAYER to consider all

 366      * @param aCornerBuffer = the buffer to store polygons

 367      * @param aInflateValue = an additionnal size to add to pad shapes

 368      *          aInflateValue = 0 to have the exact pad size

 369      * @param aMaxError = Maximum deviation from true for arcs

 370      * @param aSkipNPTHPadsWihNoCopper = if true, do not add a NPTH pad shape, if the shape has

 371      *          same size and position as the hole. Usually, these pads are not drawn on copper

 372      *          layers, because there is actually no copper

 373      *          Due to diff between layers and holes, these pads must be skipped to be sure

 374      *          there is no copper left on the board (for instance when creating Gerber Files or

 375      *          3D shapes).  Defaults to false.

 376      */
 377     void TransformPadsShapesWithClearanceToPolygon( PCB_LAYER_ID aLayer,
 378             SHAPE_POLY_SET& aCornerBuffer, int aInflateValue, int aMaxError = ARC_HIGH_DEF,
 379             bool aSkipNPTHPadsWihNoCopper = false ) const;
