 236     /**

 237      * Execute a remote command sent by Pcbnew via a socket connection.

 238      * <p>

 239      * When user selects a module or pin in Pcbnew, Eeschema shows that same

 240      * component or pin and moves cursor on the item.  The socket port used

 241      * is #KICAD_SCH_PORT_SERVICE_NUMBER which defaults to 4243.

 242      *

 243      * Valid commands are:

 244      * \li \c \$PART: \c "reference" Put cursor on component.

 245      * \li \c \$PART: \c "reference" \c \$REF: \c "ref" Put cursor on component reference.

 246      * \li \c \$PART: \c "reference" \c \$VAL: \c "value" Put cursor on component value.

 247      * \li \c \$PART: \c "reference" \c \$PAD: \c "pin name" Put cursor on the component pin.

 248      * \li \c \$NET: \c "netname" Highlight a specified net

 249      * \li \c \$CLEAR: \c "HIGHLIGHTED" Clear components highlight

 250      * <p>

 251      * They are a keyword followed by a quoted string.

 252      * @param cmdline = received command from Pcbnew

 253      */

 428     /**

 429      * Annotate the components in the schematic that are not currently annotated.

 430      *

 431      * @param aAnnotateSchematic Annotate the entire schematic if true.  Otherwise annotate

 432      *                           the current sheet only.

 433      * @param aSortOption Define the annotation order.  See #ANNOTATE_ORDER_T.

 434      * @param aAlgoOption Define the annotation style.  See #ANNOTATE_OPTION_T.

 435      * @param aStartNumber The start number for non-sheet-based annotation styles.

 436      * @param aResetAnnotation Clear any previous annotation if true.  Otherwise, keep the

 437      *                         existing component annotation.

 438      * @param aRepairTimestamps Test for and repair any duplicate time stamps if true.

 439      *                          Otherwise, keep the existing time stamps.  This option

 440      *                          could change previous annotation because time stamps are

 441      *                          used to handle annotation in complex hierarchies.

 442      * @param aLockUnits    When both aLockUnits and aResetAnnotation are true, all unit

 443      *                          associations should be kept when reannotating. That is, if

 444      *                          two components were R8A and R8B, they may become R3A and R3B,

 445      *                          but not R3A and R3C or R3C and R4D.

 446      *                          When aResetAnnotation is true but aLockUnits is false, the

 447      *                          usual behavior of annotating each part individually is

 448      *                          performed.

 449      *                          When aResetAnnotation is false, this option has no effect.

 450      * @param aReporter A sink for error messages.  Use NULL_REPORTER if you don't need errors.

 451      *

 452      * When the sheet number is used in annotation, each sheet annotation starts from sheet

 453      * number * 100.  In other words the first sheet uses 100 to 199, the second sheet uses

 454      * 200 to 299, and so on.

 455      */
 456     void AnnotateComponents( bool aAnnotateSchematic, ANNOTATE_ORDER_T aSortOption,
 457                              ANNOTATE_OPTION_T aAlgoOption, int aStartNumber,
 458                              bool aResetAnnotation, bool aRepairTimestamps, bool aLockUnits,
 459                              REPORTER& aReporter );
 460

 716     /**

 717      * Load a the KiCad schematic file \a aFileName into the sheet \a aSheet.

 718      *

 719      * If \a aSheet does not have a valid #SCH_SCREEN object, the schematic is loaded into

 720      * \a Sheet.  Otherwise, it is appended to the current #SCH_SCREEN object.

 721      *

 722      * In order to import a schematic a lot of things have to happen to before the contents

 723      * of the imported schematic can be appended to the current page.  The following list

 724      * describes this process:

 725      *

 726      * - Load the schematic into a temporary #SCH_SHEET object.

 727      * - Make sure the imported schematic does not cause any hierarchy recursion issues.

 728      * - Verify the imported schematic uses fully qualified #LIB_ID objects (symbol library table).

 729      * - Check all of the possible combinations that could cause broken symbol library links

 730      *   and give the user the option to cancel the append process.  The following conditions

 731      *   are check but they still do not guarantee that there will not be any broken symbol

 732      *   library links:

 733      *   - The source schematic is in the current project path and contains symbol library

 734      *     nicknames not found in the project symbol library table.  This can happen if the

 735      *     schematic is copied to the current project path from another project.

 736      *   - The source schematic is in a different path and there are symbol library link nicknames

 737      *     that do not exist in either the current symbol library table or the source project

 738      *     symbol library table if it exists in the source path.

 739      *   - The source schematic is in a different path and contains duplicate symbol library

 740      *     nicknames that point to different libraries.

 741      * - Check to see if any symbol libraries need to be added to the current project's symbol

 742      *   library table.  This includes:

 743      *   - Check if the symbol library already exists in the project or global symbol library

 744      *     table.

 745      *   - Convert symbol library URLS that use the ${KIPRJMOD} environment variable to absolute

 746      *     paths.  ${KIPRJMOD} will not be the same for this project.

 747      * - Clear all of the annotation in the imported schematic to prevent clashes.

 748      * - Append the objects from the temporary sheet to the current page.

 749      * - Replace any duplicate time stamps.

 750      * - Refresh the symbol library links.

 751      *

 752      * @param aSheet is the sheet to either append or load the schematic.

 753      * @param aHierarchy is the current position in the schematic hierarchy used to test for

 754      *                   possible file recursion issues.

 755      * @param aFileName is the file name to load.  The file name is expected to have an absolute

 756      *                  path.

 757      *

 758      * @return True if the schematic was imported properly.

 759      */
 760     bool LoadSheetFromFile( SCH_SHEET* aSheet, SCH_SHEET_PATH* aHierarchy,
 761                             const wxString& aFileName );
